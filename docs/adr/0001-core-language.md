# ADR 0001 — 코어 언어/런타임으로 .NET(C#) 유지

- 상태: 승인됨 (Accepted)
- 날짜: 2026-07-16
- 맥락 브랜치: `refactor/engine-extraction`

## 맥락

LyricsX-Windows를 개인용 Android 버전, 개선된 macOS 버전, 그리고 "디스플레이만 있는
브라우저 장치"(호스트가 계산한 결과만 받아 표시)로 확장하려 한다. 확장 전에
현재의 .NET 기반 코어를 유지할지, 장기 다기기 대응·성능·보안 면에서 더 적합한
언어가 있는지 결정할 필요가 있었다.

### 워크로드 성격
코어(`LyricsX.Core`)가 하는 일은 **I/O 바운드 + 경량 CPU**다.
- 4개 제공자 HTTP 호출(LRCLIB/NetEase/Kugou/QQ)
- 곡당 수 KB 텍스트에 대한 경량 암호(QQ 3중 DES 비표준 S-Box, Kugou XOR+zlib)
- LRC/KRC/QRC 파싱, 품질 점수, SQLite 캐시, DeepL 번역

고빈도 경로(글자 단위 노래방 채움)는 **코어가 아니라 각 플랫폼의 View**에 있다.
→ **성능은 언어 간 변별력이 없다.**

## 결정

**.NET(C#) 코어를 유지한다.** 그리고 경계를 언어중립 계약으로 설계해
(입력: `INowPlayingSource`, 출력: 직렬화 가능한 표시 상태) 나중에 엣지·디스플레이를
다른 언어로 붙일 여지를 남긴다.

## 근거

1. **전 타깃 도달** — .NET은 Windows(네이티브), Android(.NET for Android),
   macOS(MAUI/네이티브 interop), 브라우저(서버가 뿌리는 JSON/WebSocket을 받는
   순수 HTML/JS, 필요 시 Blazor WASM)까지 지목된 모든 기기를 커버한다.
2. **재작성 리스크 회피** — 코어의 최난도(QQ QRC 비표준 S-Box 3중 DES, Kugou KRC,
   NetEase EAPI, 과거 CDATA/속성 파싱 버그)는 이미 실제 API로 검증되고 유닛테스트
   72개가 통과한다. Rust/Kotlin 재이식은 성능 이득 없이 최고 버그 밀집 영역을
   처음부터 다시 만드는 셈이다.
3. **성능 무이득** — 위 워크로드 성격상 어떤 후보 언어도 여유롭게 처리한다.
4. **플랫폼 글루는 어차피 네이티브** — 재생감지/오버레이는 OS마다 네이티브
   (SMTC=WinRT, MediaSession=Kotlin, MediaRemote=Swift)라 코어 언어를 바꿔도 줄지 않는다.
   즉 코어 언어는 *공유 로직*에만 영향을 준다.
5. **폴리글롯은 엣지에서 선택적** — 직렬화 계약 + 네트워크 프로토콜을 두면 디스플레이는
   코어와 다른 언어여도 된다.

## 고려한 대안

| 대안 | 강점 | 채택 안 한 이유 |
|---|---|---|
| **Rust(FFI/WASM)** | 최상 성능·작은 바이너리·메모리안전 | 코어 전면 재작성(수 주+버그 재유입), 성능 이득 불필요 |
| **Kotlin Multiplatform** | Android 최강, 다플랫폼 공유 | 코어 전면 재작성, 운영 중 Windows 앱과 상충 |
| **Go / TypeScript** | 서버/브라우저 강함 | Android UI·네이티브 미디어 접근 약함, 코어 재작성 |

## 재검토(언어 전환) 트리거

아래가 **확정**되면 재검토한다(모두 "코어 재작성"이 아니라 "엣지 추가"로 해결 가능):
- Android가 사실상 유일한 장기 플랫폼이 되고 데스크톱을 접을 때 → KMP 재검토.
- 브라우저 서버가 인터넷 노출·멀티테넌트로 커질 때 → 얇은 Rust/Go 엣지를 코어 앞단에.
- Android 앱 용량이 하드 제약일 때 → 네이티브 재검토.

## 귀결

- 코어를 UI 무관 계층으로 승격: `LyricsX.Engine`(net8.0, WPF 무관)으로
  오케스트레이션을 분리하고, 소스는 `INowPlayingSource`로 추상화한다.
- 표시 상태는 직렬화 가능한 계약(향후 `PlaybackViewState`)으로 설계한다.
- 보안: 브라우저 디스플레이 서버가 새 공격면. 로컬망 한정 + 페어링 토큰 + 단방향
  브로드캐스트 + 비밀 미전파를 기본으로 한다(언어 무관, 아키텍처로 해결).
